# CAD Audit (2026-04-23)

## Findings

### High

1. `CADViewerCadEngine.tsx` was over-coupled before refactor.
- One file mixed engine bootstrap, input interaction, upload/open flow, hierarchy UI, diagnostics UI.
- Result: high regression risk and hard root-cause analysis.

2. Weak runtime typing in CadEngine boundary.
- Multiple `any` usages around `window.CadEngine`, `window.api`, scene nodes and materials.
- Result: silent runtime mismatches were possible.

### Medium

3. Sidebar hierarchy lacked entity/category-level visibility toggles.
- Layer toggles existed, but per-node hide/show controls were incomplete.

4. Long labels could overflow readable area in tree rows.
- `truncate` made long CAD labels difficult to inspect and sometimes visually clipped.

5. Diagnostics and error surface lived deep in monolithic component.
- Debugging context was available, but component complexity slowed maintenance.

### Low

6. No single source of truth document for CAD file ownership.
- Discoverability for contributors was low.

## Changes Implemented in This Round

1. Split CAD UI into dedicated modules.
- Added `CadSidebar` and `CadInspectorPanel`.
- Main viewer now orchestrates state and side effects.

2. Added typed CadEngine boundary.
- Introduced `cadEngineTypes.ts` and `runtimeBridge.ts`.
- Removed core `any` usage in key scene loading/diagnostics modules.

3. Added hierarchy visibility controls.
- Added hide/show button for entity rows and category rows in element tree.

4. Fixed long label readability in tree/list.
- Switched critical row labels to `break-all` display strategy.

5. Added collapsible left/right sidebars.
- Enables wider viewport for performance-sensitive operations.

6. Added architecture inventory documentation.
- See `docs/cad/file-inventory.md`.

7. Added unified hook contract layer and extracted document lifecycle hook.
- Added `hooks/contracts.ts` as the single source of hook I/O contracts.
- Added `useCadDocumentLifecycle` to isolate DWG open/load/close behavior from view orchestration.
- `CADViewerCadEngine.tsx` now stays focused on state composition and panel wiring.

8. Added dedicated view-state hook for UI-only concerns.
- Added `useCadViewState` to isolate layer/entity visibility, tree expansion, overlay state, and sidebar collapse.
- Reduced main component state churn and callback clutter without changing rendering behavior.

## Remaining Risks

1. `public/vendor/cadPreivew.min.js` is still a single bundled runtime.
- This round improves maintainability around it, not inside it.

2. Dimension/text completeness still depends on upstream parse fidelity and font availability.
- Must continue validating with reference DWG samples.

3. Performance parity with native desktop CAD is not expected in browser for very large drawings.
- Needs stricter progressive rendering and culling strategy in later rounds.
